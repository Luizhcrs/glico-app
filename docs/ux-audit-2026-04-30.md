# Glico — UX Audit Premium

**Data:** 2026-04-30
**Stack auditada:** Expo SDK 54, React Native 0.81, TypeScript, expo-router 6, victory-native v41
**Identidade base:** Sage Calm (#5a7a5a / #f5f3ed / #e8e4d8)
**Estado atual:** funcional, paleta consistente, sem fricção visual grave, mas "limpo demais" — falta camada de polish/microcopy/iconografia que separa MVP funcional de produto premium.

Benchmarks de referência usados como norte:
- [Apple Health](https://www.apple.com/ios/health/) — hierarquia de cards com cor semântica forte
- [Linear](https://linear.app) — tipografia Inter, microinterações conscientes, sem ruído
- [Sleep Cycle](https://www.sleepcycle.com) — tom calmo, ilustrações sutis, dados como narrativa
- [MyFitnessPal](https://www.myfitnesspal.com) Premium — empty states cuidados, segmented controls, gráficos interativos
- [Streaks](https://streaksapp.com) — tipografia variável + iconografia SF Symbols / Phosphor

## 1. Tipografia

| ID | Problema | Onde | Referência | Esforço | Impacto |
|----|----------|------|------------|---------|---------|
| T1 | Apenas font system. Em iOS vira SF, em Android vira Roboto — quebra a identidade visual entre plataformas | tudo | Linear (Inter), Apple Health (SF), Notion (Inter) | 1.5h | **Alto** |
| T2 | Hero number (`BigNumber`) usa weight 200 system — em Android Roboto Light fica fininho demais e perde presença | `src/ui/components/BigNumber.tsx:36` | Apple Health usa SF Pro Display weight 300 com letterSpacing -2 | 0.5h | **Alto** |
| T3 | Sem escala tipográfica explícita (display / headline / title / body / caption). Os tamanhos `xs/sm/md/lg/xl/hero` cobrem só size, não pareiam line-height + weight + spacing | `src/ui/theme.ts:24` | Apple HIG typography scale, Material 3 type system | 1h | Médio |
| T4 | Numbers não usam font tabular consistente. `BigNumber` tem `fontVariant: ['tabular-nums']` mas as listas (`rowValue`, `units` em insulin) não — ao mudar o número, o layout pula | `app/index.tsx:183`, `app/insulin.tsx:81` | qualquer app financeiro premium (Revolut, Robinhood) | 0.3h | Médio |
| T5 | Letras nada padronizadas: `letterSpacing` aparece como -0.5, -1.5, 0.2, 0.3, 0.6, 0.8, 1 em arquivos diferentes — falta token | múltiplos | design tokens canônicos (Radix, Tailwind) | 0.3h | Baixo |
| T6 | Títulos usam tamanho 28 hardcoded em vez de token (`fontSizes.hero` é 56, `xl` é 24, falta 28) | `app/index.tsx:130`, `app/onboarding/welcome.tsx:42`, etc | escala uniforme | 0.2h | Baixo |

**Recomendação canônica:** instalar [Plus Jakarta Sans](https://fonts.google.com/specimen/Plus+Jakarta+Sans) (weights 400/500/600/700) ou [Inter](https://rsms.me/inter/) via `expo-font`. Plus Jakarta tem personalidade humana mais alinhada com Sage Calm; Inter é mais neutra/profissional.

## Notas finais

- **Fundação está sólida**: paleta Sage Calm é consistente, copy em PT-BR é humana, telas funcionam. O gap não é estrutural — é camada de polish e detalhamento.
- **Top 3 itens de impacto desproporcional**: tipografia (T1), iconografia (I1), feedback ao salvar/toast (S5+S6+F1). Esses 3 sozinhos transformam a percepção sem mexer em arquitetura.
- **Cuidado com over-design**: usuário rejeitou animações decorativas. Tudo que está marcado microinteração nesse audit é funcional (haptic, feedback de save, swipe-to-delete) — nada decorativo.
- **Itens marcados "fase 2" no spec** (CGM, caregiver, dark mode, acessibilidade WCAG completa) propositalmente ficam fora do roadmap premium imediato.
